Welcome to the AI-space corner of this site. This first post is intentionally a
**template** — it shows every building block available to any future post. Copy
it, change the frontmatter, replace the content, and ship.

## What you can write with

Everything here is just **MDX** — Markdown plus a handful of React components
that keep the styling consistent across posts.

### Prose, the way you'd expect

You get **bold**, _italics_, `inline code`, [links](https://react.dev),
bulleted lists, numbered lists, tables, and block quotes. Nothing fancy:

- Use Markdown for 90 % of the text.
- Reach for JSX components when you need rich media or callouts.
- Keep paragraphs short. Respect the reader's time.

> "Any fool can write code that a computer can understand. Good programmers
> write code that humans can understand." — Martin Fowler

### Code with syntax highlighting

Fenced code blocks get highlighted at **build time** (no runtime JS cost):

```ts
type Agent = {
  name: string;
  tools: string[];
  run: (prompt: string) => Promise<string>;
};

async function chain(agents: Agent[], input: string) {
  let output = input;
  for (const agent of agents) {
    output = await agent.run(output);
  }
  return output;
}
```

### Callouts

Use callouts sparingly, for real asides:

<Callout variant="info" title="Heads up">
  This is an informational callout — good for context the reader can skip.
</Callout>

<Callout variant="tip" title="Pro tip">
  Prefer small, frequent posts over giant comprehensive ones. Momentum &gt; perfection.
</Callout>

<Callout variant="warn" title="Careful">
  Anything time-sensitive (model versions, API shapes) will rot fast. Date the post clearly.
</Callout>

## That's it

Copy this file, rename it to `YYYY-MM-DD-your-slug.mdx`, swap out the
frontmatter and body, and you're live the moment the build finishes.
